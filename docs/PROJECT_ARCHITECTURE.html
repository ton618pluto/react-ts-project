⭐面试亮点总结（详细实现）

1. 低代码编辑器设计

核心思路：组件注册 + 动态渲染

  • 每个组件通过 default.ts 导出 ComponentConfType 配置对象
  • 统一入口 questionComponents/index.ts 提供 getComponentConfByType(type) 函数
  • EditCanvas 通过该函数动态获取并渲染组件
// 组件配置 (default.ts) const QuestionTitleConf: ComponentConfType = { title: '标题', type: 'questionTitle', Component: QuestionTitleComponent, PropComponent: TitlePropComponent, defaultProps: QuestionTitleDefault, } // 动态渲染 (EditCanvas.tsx) const componentConf = getComponentConfByType(type) return <componentConf.Component {...props} />

2. 撤销/重做功能

核心思路：redux-undo 包装 + 过滤不相关 action

  • 使用 undoable(componentStore, { limit: 20 }) 包装
  • 使用 excludeAction 过滤：resetComponents、changeSelectedId、selectPreComponent、selectNextComponent
  • 只保留实际数据变更（添加/删除/修改组件）到撤销历史
被过滤的 Action原因
resetComponents整体替换，不应纳入撤销
changeSelectedIdUI状态变化是瞬时的
selectPre/NextComponent导航操作是瞬时的

3. 拖拽排序实现

核心思路：@dnd-kit + Redux 状态更新

  • 封装 SortableContainer 和 SortableItem 组件
  • 使用 useSortable 和 SortableContext
  • 拖拽结束 dispatch moveComponentPosition 更新顺序

4. 组件化设计（可扩展）

📝面试快速回答模板

Q1: 请介绍一下你做过的项目

我开发的是一个问卷调查/低代码平台，支持问卷的创建、编辑、发布和统计分析。核心技术栈是 React 19 + TypeScript + Redux Toolkit + Ant Design。

Q2: 项目的整体架构是怎么样的？

项目采用典型的 React + Redux 架构。目录结构分为 components（组件）、pages（页面）、store（状态）、services（API）等模块。

  • 路由层：Hash 路由 + 懒加载
  • 状态层：Redux Toolkit 管理用户信息、问卷组件、页面信息，其中问卷组件使用 redux-undo 支持撤销/重做
  • 视图层：Ant Design 组件库，7种问卷组件采用统一规范设计

核心亮点是低代码编辑器，通过"组件注册 + 动态渲染"的配置化设计，支持拖拽排序、属性编辑、复制粘贴等功能。

Q3: 你觉得这个项目的亮点是什么？

  1. 低代码编辑器设计：采用配置化思路，每个问卷组件导出配置对象，通过 getComponentConfByType 动态渲染。新增组件非常方便。
  2. 撤销/重做功能：使用 redux-undo 实现，限制20步，并且过滤掉了选中状态、导航操作等不相关的 action，只保留实际数据变更。
  3. 拖拽排序：使用 @dnd-kit 实现，封装了 SortableContainer 和 SortableItem 组件，在画布和图层面板都支持拖拽。
  4. 性能优化：路由懒加载，每个页面独立打包成 chunk，减少首屏加载时间。
  5. TypeScript：有完整的类型定义，组件配置、props 等都有明确的类型约束。

Q4: 状态管理是怎么设计的？

使用 Redux Toolkit，主要有三个 store 模块：

  • userInfo：用户登录信息
  • components：问卷组件列表，用 undoable 包装支持撤销/重做
  • pageInfo：页面标题、描述等基本信息

components 是核心，包含 addComponent、changeComponentProp、deleteComponent、moveComponentPosition 等 action，支持组件的增删改和拖拽排序。

我们采用了配置化 + TypeScript 类型约束的组件设计。每个问卷组件都遵循统一的目录结构，包含4个文件：index.tsx（渲染）、default.ts（配置）、PropComponent.tsx（属性编辑）、StatComponent.tsx（统计）。通过 ComponentConfType 接口定义组件配置对象，使用 getComponentConfByType 函数根据 type 动态获取。这样编辑器、属性面板、统计页面都能复用同一套组件逻辑，新增组件也非常方便，只需要创建文件夹并注册配置即可。

Q6: 如何实现撤销/重做的？

使用 redux-undo 库。用 undoable 包装 componentsReducer，配置最大撤销20步。同时使用 excludeAction 过滤掉 resetComponents（重置）、changeSelectedId（选中）、selectPreComponent/selectNextComponent（导航）等不相关的 action，这些是瞬时 UI 状态，不需要纳入撤销历史。实际的数据变更（添加/删除/修改组件）都会被记录，用户可以安心尝试各种编辑操作。

Q7: 如何基于 @dnd-kit 实现拖拽排序的？

1. 拖拽实现架构

项目使用 @dnd-kit 实现拖拽功能，主要在以下3个场景使用：

  • 📝 EditCanvas（画布区）：问卷组件在画布中拖拽排序
  • 📋 Layers（图层区）：图层列表中拖拽排序
  • 📦 ComponentLib（物料区）：点击添加（不是拖拽）

2. 封装拖拽容器组件

// components/DragSortable/SortableContainer.tsx
import { DndContext, MouseSensor, useSensor, useSensors, DragEndEvent } from '@dnd-kit/core'
import { SortableContext, verticalListSortingStrategy } from '@dnd-kit/sortable'

type PropsType = {
  children: JSX.Element | JSX.Element[]
  items: Array<{ id: string; [key: string]: any }>
  onDragEnd: (oldIndex: number, newIndex: number) => void
}

const SortableContainer: FC = ({ children, items, onDragEnd }) => {
  // 配置鼠标传感器，8px 移动才触发拖拽
  const sensors = useSensors(
    useSensor(MouseSensor, { activationConstraint: { distance: 8 } })
  )

  // 拖拽结束处理
  function handleDragEnd(event: DragEndEvent) {
    const { active, over } = event
    if (over === null) return
    if (active.id !== over.id) {
      const oldIndex = items.findIndex(item => item.fe_id === active.id)
      const newIndex = items.findIndex(item => item.fe_id === over.id)
      onDragEnd(oldIndex, newIndex)
    }
  }

  return (
    <DndContext sensors={sensors} collisionDetection={closestCenter} onDragEnd={handleDragEnd}>
      <SortableContext items={items} strategy={verticalListSortingStrategy}>
        {children}
      </SortableContext>
    </DndContext>
  )
}

3. 封装拖拽项组件

// components/DragSortable/SortableItem.tsx
import { useSortable } from '@dnd-kit/sortable'
import { CSS } from '@dnd-kit/utilities'

type ItemProps = {
  id: string
  children: JSX.Element
}

const SortableItem: FC = ({ children, id }) => {
  const { attributes, listeners, setNodeRef, transform, transition } = useSortable({ id })

  const style = {
    transform: CSS.Transform.toString(transform),  // 拖拽时的位移
    transition,  // 过渡动画
  }

  return (
    <div ref={setNodeRef} style={style} {...attributes} {...listeners}>
      {children}
    </div>
  )
}

4. 画布中使用拖拽（EditCanvas.tsx）

// pages/question/Edit/EditCanvas.tsx
import SortableContainer from '@/components/DragSortable/SortableContainer'
import SortableItem from '@/components/DragSortable/SortableItem'
import { useAppDispatch } from '@/store/types'
import { moveComponentPosition } from '@/store/modules/componentsReducer'

const EditCanvas: FC = () => {
  const { componentsList } = useGetComponentInfo()
  const dispatch = useAppDispatch()

  // 处理拖拽结束，更新 Redux 状态
  function handleDragEnd(oldIndex: number, newIndex: number) {
    dispatch(moveComponentPosition({ oldIndex, newIndex }))
  }

  // 转换为 dnd-kit 需要的 id 格式
  const componentsListWithId = componentsList.map(comp => ({
    id: comp.fe_id,
    ...comp,
  }))

  return (
    <SortableContainer items={componentsListWithId} onDragEnd={handleDragEnd}>
      <div className={styles.canvas}>
        {componentsList.filter(c => !c.isHidden).map(item => (
          <SortableItem key={item.fe_id} id={item.fe_id}>
            <div className={...}>
              {getComponent(item)}
            </div>
          </SortableItem>
        ))}
      </div>
    </SortableContainer>
  )
}

5. Redux 中处理拖拽排序

// store/modules/componentsReducer/index.ts
// 拖拽排序 action
export const moveComponentPosition = (payload: { oldIndex: number; newIndex: number }) => ({
  type: MOVE_COMPONENT_POSITION,
  payload,
})

// reducer 中处理
case MOVE_COMPONENT_POSITION: {
  const { oldIndex, newIndex } = action.payload
  const newComponents = [...state.componentsList]
  const [removed] = newComponents.splice(oldIndex, 1)  // 取出旧位置
  newComponents.splice(newIndex, 0, removed)  // 插入新位置
  return { ...state, componentsList: newComponents }
}

6. 核心原理总结

dnd-kit 核心概念：

  • DndContext：拖拽上下文，包裹整个拖拽区域
  • SortableContext：可排序列表上下文，管理所有可拖拽项
  • useSortable：Hook，让元素变得可拖拽，返回 ref、style、listeners
  • MouseSensor：鼠标传感器，配置 distance: 8 防止误触
  • DragEndEvent：拖拽结束事件，包含 active（起始）和 over（结束）信息

面试话术：

我们封装了 SortableContainer 和 SortableItem 两个组件。Container 使用 DndContext 包裹，内部用 SortableContext 管理可排序列表；Item 使用 useSortable hook 让元素可拖拽。拖拽结束时通过 onDragEnd 回调拿到 oldIndex 和 newIndex，然后 dispatch moveComponentPosition 更新 Redux 中的 componentsList 数组。这种方式在画布和图层区都复用了同一套逻辑。
